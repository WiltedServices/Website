▶Swapped exploit (getting admin)

Also known as Sticky Keys/Utilman exploit.

  • Access an elevated Command Prompt/Powershell instance. There are multiple ways to do this:
    • Boot into Windows RE, selecting "Advanced Options", then "Command Prompt"
    • Booting a Windows installation media on an external storage device & selecting Next > Repair Your Computer > Troubleshoot > Command Prompt
  • Run “C:”
    • Note that Windows can be installed on any drive, C is just the most common one.
  • Run “dir”. Keep going until you find a drive with the Windows directory (1).
  • Create a backup by running “copy sethc.exe sethc.bak” for Sticky Keys or "copy utilman.exe utilman.bak" for Utilman.
    • Note other files can be swapped, these are just the most common ones.
  • Swap the respective file with another file.
    • Swapping with Command Prompt; run “copy cmd.exe sethc.exe” for Sticky Keys or "copy cmd.exe utilman.exe" for Utilman.
  • Type "Y" to proceed.
  • Activate Sticky Keys (press Shift 5 times) or click the Ease of Access button (on the login screen as well as Ctrl+Alt+Delete).
  • Run any commands.
▶Temporarily Removing Family Safety
  • Open the task manager
  • Go to the details tab
  • Find or search for "wpcmon.exe"
  • Right-click, and hit “End process tree”
    • if you can't end it because you don't have admin, use the Swapped exploit
    • the application will start back up if you shut off your device or put it to sleep if you want to fix this, find Ending Family Safety on startup
▶Ending Family Safety on Startup
  • On your Windows device, press Windows+R
  • Enter services.msc into the text box that appears and hit enter
  • In services, find “Parental Controls” and right-click and select properties
  • In the properties popup, turn the startup status to “disabled”

If you would like to revert this, do all the same steps but change the startup status to automatic

▶Permanently removing Family Safety
  • On your Windows device, find wpcmon.exe (can be done in many ways, such as the Run dialog or Task Manager, located in System32), then delete it.
